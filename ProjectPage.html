Event System

During development we noticed that a lot of interactions needed events to run or to communicate with a different behaviour. So I made this system to make it easier to quickly add new events to gameobjects. It works by having an event listener which checks the event it needs to play and link that with a key. This event listener automatically adds itself to the dictionary in the event manager. Finally, there is an event invoker which in our case was most often a collider.

Project description

For this 8-week school project we had to make a first person puzzle game.

The first two weeks were about pre-production, in this phase we had to come up with a main mechanic, present our ideas, design levels around this mechanic, and build a proof of concept.

After that we started working on production for the next 6 weeks were the primary focus was on creating a polished game.
