SCRUM Master:

Every group project we've done so far we've made use of SCRUM in our workflow. For these projects I've often taken up the role of Scrum master. Within my role I've set up scrum boards, lead sprint reviews and retrospectives and set up clear boundaries with the team so we can make sure we'll have a great project in the end.
I operate from the mindset that our main focus should always be to finish the project on time. One of the main rules I always try to implement is that when one of the team members is lacking behind I encourage myself and the team to refrain from judging them. Whether a reason for slacking behind is valid or not is not up to me to decide so I won't, instead I'll focus on how we can pick up the work that's been left behind.
